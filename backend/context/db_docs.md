# Borneo Indigenous Language Preservation Platform
## Database Documentation

## 1. Overview

This document describes the **database architecture** for the **AI Indigenous Language Revitalization Platform**. The system is designed to help communities record, preserve, and learn endangered languages through digital archiving and AI-assisted learning tools.

The backend uses:

- **Database:** MongoDB
- **File Storage:** Supabase Storage
- **Backend API:** FastAPI
- **Data Format:** JSON / BSON

The database stores **metadata and structured data**, while large files such as **audio recordings** are stored in **Supabase Storage**.

---

## 2. Database Architecture

### Database Name

```
borneo_language_archive
```

### Collections

The system contains four main collections:

```
users
recordings
stories
lessons
```

Each collection stores a specific type of data required by the system.

---

## 3. Collection Relationships

MongoDB does not enforce foreign keys, but relationships are implemented using **ObjectId references**.

```
users._id
↓
recordings.userId

users._id
↓
stories.createdBy
```

Relationship diagram:

```
    users
      │
      │ _id
      │
┌────────┴────────┐
│                 │
recordings      stories
userId        createdBy
```

Meaning:

- A **user** can upload many **recordings**
- A **user** can create many **stories**

## 4. Users Collection

### Purpose

Stores information about platform users.

Users can have different roles:

```
elder
learner
admin
```

### Schema

| Field | Type | Description |
|-------|------|-------------|
| _id | ObjectId | Unique identifier generated by MongoDB |
| name | String | User's name |
| email | String | Login email address |
| passwordHash | String | Hashed password |
| role | String | User role (elder, learner, admin) |
| points | Integer | Gamification score |
| badges | Array[String] | Achievement badges |
| languagePreference | String / null | Preferred learning language |
| createdAt | Date | Account creation timestamp |
| lastLogin | Date / null | Last login time |
| status | String | Account status (active / suspended) |

### Example Document

```json
{
  "_id": ObjectId("69ad64ac7f4c23fe225b86eb"),
  "name": "John Doe",
  "email": "john@example.com",
  "passwordHash": "hashed_password",
  "role": "elder",
  "points": 10,
  "badges": ["storyteller"],
  "languagePreference": "iban",
  "createdAt": ISODate("2026-03-08T12:00:00Z"),
  "lastLogin": null,
  "status": "active"
}
```

### Indexes

| Index | Type | Purpose |
|-------|------|---------|
| _id | Unique | Default MongoDB primary key |
| email | Unique | Prevent duplicate accounts |
| role | Normal Index | Faster role-based queries |

Example queries:

```
find users by role
find user by email
```

## 5. Recordings Collection

### Purpose

Stores metadata for **elder audio recordings**.

Audio files themselves are stored in **Supabase Storage**, while MongoDB stores:

- audio URL
- transcript
- metadata
- privacy settings

### Schema

| Field | Type | Description |
|-------|------|-------------|
| _id | ObjectId | Recording ID |
| userId | ObjectId | Reference to users._id |
| audioUrl | String | Supabase storage URL |
| transcript | String | Speech-to-text result |
| language | String | Language spoken |
| duration | Integer | Audio duration (seconds) |
| consent | Boolean | Elder consent for recording |
| visibility | String | Access level |
| aiProcessed | Boolean | Whether AI processing completed |
| createdAt | Date | Recording timestamp |

### Visibility Values

```
private
community
public
```

| Visibility | Access |
|------------|--------|
| private | uploader only |
| community | registered learners |
| public | everyone |

### Example Document

```json
{
  "_id": ObjectId("69ad6f627f4c23fe225b86f7"),
  "userId": ObjectId("69ad64ac7f4c23fe225b86eb"),
  "audioUrl": "https://xxxxx.supabase.co/storage/v1/object/public/recordings/elder_story_001.mp3",
  "transcript": "Long ago in the forest...",
  "language": "iban",
  "duration": 45,
  "consent": true,
  "visibility": "community",
  "aiProcessed": true,
  "createdAt": ISODate("2026-03-08T12:00:00Z")
}
```

### Indexes

| Field | Purpose |
|-------|---------|
| userId | retrieve recordings by elder |
| language | language-based filtering |
| createdAt | sort recordings by time |

Example query:

```
get all recordings by elder
get recordings in a specific language
```

## 6. Stories Collection

### Purpose

Stores **traditional stories and folklore** contributed by elders.

Each story may contain:

- original story text
- children's simplified version
- narration audio
- cultural tags

### Schema

| Field | Type | Description |
|-------|------|-------------|
| _id | ObjectId | Story ID |
| title | String | Story title |
| language | String | Language used |
| text | String | Original story |
| childrenVersion | String | Simplified children's version |
| audioUrl | String | Narration audio |
| createdBy | ObjectId | Reference to users._id |
| tags | Array[String] | Story categories |
| createdAt | Date | Creation timestamp |

### Example Document

```json
{
  "_id": ObjectId("69ad6f627f4c23fe225b8700"),
  "title": "The Forest Spirit",
  "language": "iban",
  "text": "Long ago the forest was protected by a spirit...",
  "childrenVersion": "A friendly forest spirit protected the village.",
  "audioUrl": "https://xxxxx.supabase.co/storage/v1/object/public/stories/story_audio_001.mp3",
  "createdBy": ObjectId("69ad64ac7f4c23fe225b86eb"),
  "tags": ["folklore", "culture"],
  "createdAt": ISODate("2026-03-08T12:00:00Z")
}
```

### Indexes

| Field | Purpose |
|-------|---------|
| language | language filtering |
| tags | category filtering |
| createdAt | chronological sorting |

Example queries:

```
get stories by language
get folklore stories
```

## 7. Lessons Collection

### Purpose

Stores **language learning materials** used in the learning module.

Lessons include:

- vocabulary lists
- pronunciation audio
- quizzes
- difficulty level

### Schema

| Field | Type | Description |
|-------|------|-------------|
| _id | ObjectId | Lesson ID |
| title | String | Lesson title |
| category | String | Lesson category |
| difficulty | String | Learning difficulty |
| language | String | Language taught |
| vocabulary | Array | Vocabulary list |
| quiz | Array | Quiz questions |
| createdAt | Date | Creation timestamp |

### Difficulty Levels

```
beginner
intermediate
advanced
```

### Vocabulary Structure

| Field | Type | Description |
|-------|------|-------------|
| word | String | Indigenous word |
| translation | String | Translation |
| audioUrl | String | Pronunciation audio |

### Quiz Structure

| Field | Type | Description |
|-------|------|-------------|
| question | String | Quiz question |
| options | Array[String] | Multiple choice options |
| answer | String | Correct answer |

### Example Document

```json

  "_id": ObjectId("69ad6f627f4c23fe225b8710"),

  "createdAt": ISODate("2026-03-08T12:00:00Z")
```

## 8. Supabase Storage Integration

Audio files are stored in **Supabase Storage**.

MongoDB stores only the **file URL**.

### Storage Structure

```
supabase-storage
│
├── recordings
│   └── elder_story_001.mp3
│
├── stories
│   └── story_audio_001.mp3
│
└── lessons
    └── vocab_audio_001.mp3
```

### Data Flow

```
User records audio
      ↓
Frontend uploads file
      ↓
Supabase Storage
      ↓
Returns file URL
      ↓
FastAPI backend
      ↓
MongoDB stores metadata
```
